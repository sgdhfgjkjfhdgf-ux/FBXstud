# General guidelines

* Всегда думай «система, а не один экран».
* Дизайн должен быть **super-modern**: много воздуха, крупная типографика,
  аккуратная сетка, минимализм, фокус на контенте.
* По умолчанию используй **компоненты, токены и стили из Align Kit**.
  Не изобретай новые, если задачу можно решить готовым компонентом.
* Макеты должны быть **responsive-ready**:
  - desktop: 1440 px (основной),
  - tablet: 768 px,
  - mobile: 390 px.
* Избегай декоративного хаоса: максимум 1–2 визуальных акцента на экран.
* Все элементы интерфейса должны быть выровнены по сетке и опорным отступам
  Align Kit.

# Layout & grid

* Используй 12-колоночную сетку для desktop и 4–6 колонок для мобильных.
* Основные паттерны:
  - sticky top-nav,
  - hero-блок с двумя колонками (текст + иллюстрация/карточки),
  - секции с чёткими блоками и визуальными разделителями.
* Вертикальный ритм — кратный базовому шагу из Align Kit (например 8 px).
* Не используй абсолютное позиционирование, если без него можно обойтись.

# Design system guidelines (Align Kit)

Всегда начинай с Align Kit:

* Используй **готовые стили текста** (headline, subtitle, body, caption).
* Используй **готовые цветовые токены** (bg, surface, border, text, subtle).
* Применяй **компоненты**:
  - кнопки,
  - поля ввода,
  - dropdown, tabs, chips,
  - карточки, модалки, навигация, таблицы.

Не создавай кастомные компоненты, пока:
1) не попробуешь составить экран из существующих,
2) не убедишься, что кейс действительно новый.

## Typography

* Базовый размер шрифта: **14 px** для body, 16 px для важных текстов.
* Заголовки — крупные, контрастные, с чёткой иерархией (H1 > H2 > H3).
* Никогда не используй больше 3 уровней размера шрифта на одном экране.
* Не используй капс без надобности — только для key-labels и маленьких тегов.

## Color & theming

* Основная палитра:
  - светлый нейтральный фон,
  - контрастный текст,
  - акцентные элементы через токены Align Kit (primary / accent).
* Избегай пёстрых градиентов и случайных цветов.
* Для акцентов используй либо один основной цвет, либо его вариации по тону.

## Buttons

* Primary Button:
  - используется для **единственного главного действия** на блок/экран,
  - всегда визуально доминирует над остальными.
* Secondary Button:
  - для альтернативного действия рядом с primary.
* Tertiary / Ghost:
  - для вспомогательных действий (например, «Подробнее»).

Не ставь две Primary кнопки рядом. Если действий два — одна Primary, одна Secondary.

# Interaction & motion

* Анимации — быстрые, мягкие, с плавным easing.
* Используй микро-анимации:
  - hover на кнопках,
  - лёгкие переходы между вкладками,
  - subtle fade/slide при появлении модалок.
* Не перегружай экраны: одна-две анимации на блок достаточно.

--------------

# Content & structure

* Тексты должны быть:
  - короткими,
  - конкретными,
  - ориентированными на действие.
* В заголовках — ценность, в подзаголовках — контекст.
* Избегай lorem ipsum там, где можно сгенерировать осмысленный текст,
  поясняющий функциональность интерфейса.

--------------

# Components usage rules

* Приоритизируй:
  1. Готовый компонент из Align Kit.
  2. Вариацию существующего компонента (через props/variants).
  3. Новый компонент — только если 1 и 2 не подходят.

* Панели/карточки:
  - используем для группировки логически связанного контента,
  - всегда с понятным заголовком / label.

* Формы:
  - поля ввода выравнивать по сетке,
  - четкая иерархия: label → поле → helper/error text,
  - минимальная когнитивная нагрузка: группируй поля.

--------------

# Patterns & UX

* Не используем dropdown, если опций ≤ 2 — лучше переключатель или радиокнопки.
* Bottom toolbar:
  - максимум 4 иконки/пункта,
  - не использовать вместе с floating action button.
* Chips/tags:
  - использовать наборами по 3 и более,
  - для фильтрации или быстрой сегментации.

--------------

# Accessibility

* Контраст текста и фона — не ниже WCAG AA.
* Размер интерактивных областей — не менее 44×44 px.
* Всегда учитывай состояние:
  - default,
  - hover,
  - active,
  - disabled,
  - error.

--------------

# Figma Make specific

* Всегда явно указывай, что нужно использовать **Align Kit**:
  - «Use Align Kit components and tokens wherever possible».
* При генерации экранов проси:
  - создать сетку,
  - применить стили из Align Kit,
  - структуру в Auto-layout.
* Если создаются новые компоненты — они должны быть совместимы по стилю
  с Align Kit и использовать те же токены.
